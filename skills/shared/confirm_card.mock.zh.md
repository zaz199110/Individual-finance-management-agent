## 六 · `report_publish`（RPT-CARD-01 · 四报告统一）

**位置**：模式 B · **右侧聊天列**（非 Preview 内）。

**卡片正文（示例 · 需求梳理）**

```text
《张先生 · 退休养老 · 投资需求报告》草稿已就绪
完整内容请在左侧预览区查看。

[ 确认发布 ]  [ 放弃草稿 ]
```

**确认发布后助手一句**：已保存至「我的报告 · 投资需求」。

**放弃草稿 · 投资需求报告专用追问**（PH-PROFILE-UNDO-02 · **紧接** 上卡 · **非** `report_publish` 卡）

```text
您刚才放弃了这份报告草稿；基本情况/本组目标的修改已经保存，但还没有通过投资需求报告完成确认发布。

相对上一份已确认的投资需求（{日期}），目前变了这些：
· 税后年收入：80 万 → 95 万
· 目标年限：2035 → 2038
…

是否一并放弃这些修改，恢复为上一份已确认的内容？
若只是报告措辞要调整，不必放弃修改，直接在对话里说明即可。

[ 继续完成投资需求确认 ]  [ 放弃这些修改，恢复上一版 ]
```

**按钮语义**

| 按钮 | 行为 |
|------|------|
| **继续完成投资需求确认** | 不回滚写库 · 须续接 §6.2.8 直至 **确认发布** |
| **放弃这些修改，恢复上一版** | **有** 上一份已发布报告 · 整包回滚至旧修订 |
| **取消本次新建** | **无** 已发布报告（首次建组）· **停用** 该组 · **不** 假装还原 |

**无旧报告 · 追问卡正文（PH-PROFILE-UNDO-03）**

```text
您刚才放弃了这份报告草稿；「{场景名}」约束已保存，但还没有任何确认发布过的投资需求报告，
因此没有「上一版」可以恢复。

若不想继续这一组，可以取消本次新建（该组将停用，不再催您发报告）。
若只是报告措辞要改，不必取消，直接在对话里说明即可。

[ 继续完成投资需求确认 ]  [ 取消本次新建 ]
```

## Agent 须知（不对客）

- `summary_zh`（≤120 字）给侧栏/通知用，**不能**替代卡片正文。
- 修订一律 **新 propose**；旧卡 `superseded`（P-04）。
