# 智能投顾助手

> C 端投资者自有的中立基金理财工具 — 专注基金研究和配置，不做交易。

---

## 产品定位

**为什么需要自有的理财工具？**

- 平台有营销立场，推荐不中立
- 资产分散在多个平台，无法统筹
- 你需要一个中立工具用于研究、记录、规划

**这个工具做什么？**

- 客观数据研究你感兴趣的产品
- 按你的需求推荐基金
- 帮你确定资产配置方案

**核心原则：** 工具不做交易，专注基金研究和配置。用户交易选择费率最低渠道即可。

---

## 产品亮点

### 1. 基金解读：四层数据瀑布

不是调 API，是建引擎。从行情数据到联网验证，逐级下探，来源可追溯。

| 层级 | 数据源 | 说明 |
|------|--------|------|
| L0 | 行情数据 | 基金净值、涨跌、规模等基础数据 |
| L1 | 招募书披露 | 9 个预定义查询（费率/风险/策略/分红…）保底安全网 |
| L2 | 语义检索 | 关键词反向增强 L1 检索 |
| L3 | 联网验证 | 公开资讯补充验证 |

**设计原则：** LLM 仅整合，不生成判断。不可验证 = 不可用。

### 2. Harness 框架：五层控制中枢

- **Planner** — LLM 意图分析
- **Planner Rules** — 硬规则纠偏，防止模型误判
- **工作流锁** — 防并发污染
- **场景分发** — 5 个场景模块
- **SSE 流式推送** — 阶段式输出，心理等候感可控

### 3. 上下文管理：四层压缩 + 业务锚点还原

执行顺序固定 L3→L1→L2→L4，确保重要信息不丢失。业务锚点（持仓快照、基金代码、任务阶段、用户确认状态）从数据库恢复，永不丢失。

### 4. 安全兜底机制

- **规则纠偏** — 防止模型误判（如基金代码被识别为简单问答）
- **工作流锁** — 防止并发污染，保证会话状态一致

## 产品功能

五个功能形成闭环，覆盖五个投资场景。

### 核心功能

| 功能 | 说明 |
|------|------|
| **自由问答** | 想知道什么直接问，支持联网搜索和截图提问 |
| **需求梳理** | 确定你的风险偏好和财务目标，生成投资需求报告 |
| **资产配置** | 基于需求报告，两步生成方案：先定大类资产比例，再选具体基金 |
| **持仓诊断** | 录入现有持仓，系统自动汇总分析，生成持仓诊断报告 |
| **基金解读** | 输入基金代码，获得从行情到深层语义的多层次分析 |

### 投资场景

- 结婚生育
- 购房置业
- 子女教育
- 退休养老
- 财富增值

### 输出报告

四类报告经历「草稿 → 核验 → 确认 → 发布归档」完整流程，可回顾、可归档。

## 系统架构

```
用户消息 → 意图分析 → 场景分发 → 业务数据调用 → 模型推理 → 报告输出
```

**四层结构：**

1. **场景模块** — 自由问答 / 需求梳理 / 资产配置 / 持仓诊断 / 基金解读
2. **业务数据** — 用户画像、资产配置方案、持仓快照、知识库
3. **基础设施** — 多模型槽（推理/深度推理/视觉/联网/向量）、Supabase 数据库
4. **输出报告** — 投资需求报告、资产配置方案、持仓分析报告、基金解读报告

## 部署指南

### 环境要求

| 依赖 | 最低版本 | 说明 |
|------|----------|------|
| Node.js | 18+（推荐 20+） | 运行环境 |
| npm | 9+ | 包管理器 |
| Docker Desktop | 最新稳定版 | 仅本地演示模式需要 |
| Python | 3.9+ | 可选，高级功能需要 |

### 部署模式

| 模式 | 适用场景 | 需要 Docker | 上手时间 |
|------|----------|-------------|----------|
| 本地演示 | 本机演示、产品验收 | 是 | 约 5-10 分钟 |
| 云端数据库 | 已有 Supabase 云端项目 | 否 | 约 3-5 分钟 |

### 快速开始

```bash
# 1. 复制配置文件
cp .env.example .env.local
# 编辑 .env.local 填入你的 API 密钥

# 2. 安装依赖
npm install --legacy-peer-deps

# 3. 启动数据库（本地演示模式，需 Docker）
npm run supabase:start

# 4. 同步设置
npm run supabase:sync-settings

# 5. 初始化数据
npm run data:init

# 6. 启动开发服务器
npm run dev
```

应用启动后访问 http://localhost:3000

### 环境变量配置

在 `.env.local` 中配置以下变量：

```bash
# Supabase 配置
SUPABASE_URL=http://127.0.0.1:54321  # 本地演示
# SUPABASE_URL=https://your-project.supabase.co  # 云端
SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key

# 大语言模型
LLM_API_KEY=your-deepseek-key
LLM_MODEL_NAME=deepseek-chat

# 应用配置
APP_ENV=development
APP_SECRET=change-me-to-a-random-string
```

详细配置说明见 [.env.example](.env.example)
