# 对客正文（复制发送）

您的基本信息已存在。以下列出当前数据，如需修改请在下方按格式回复。

**当前数据：**
- 姓名：{name}
- 年龄：{age} 岁
- 家庭状况：{marital_status_label}
- 职业：{occupation_label}
- 税后年收入：{annual_income_after_tax} 元
- 月税后到手：{monthly_income_after_tax} 元
- 可投资金融资产：{financial_assets} 元
- 贷款待还总额：{loan_balance_total} 元
- 月还贷现金：{monthly_loan_payment} 元
- 月固定支出：{monthly_fixed_expense} 元
- 月可投资：{monthly_investable} 元

---

如需修改，请按以下格式回复（只写有变化的项目）：

1. 姓名/称呼（自由填写）

2. 年龄（填写数字，单位：岁）

3. 家庭状况（选字母）：
   A. 单身，无子女
   B. 已婚，无子女
   C. 已婚，有1个孩子
   D. 已婚，有2个孩子
   E. 离异
   F. 自填（请描述家庭情况）

4. 职业（选字母）：
   A. 企业员工
   B. 自由职业
   C. 公务员/事业单位
   D. 企业主/做生意
   E. 退休
   F. 自填（请描述职业）

5. 税后年收入（填写数字，单位：元/年）

6. 每月税后到手收入（填写数字，单位：元/月）

7. 目前可投资的金融资产总额（填写数字，单位：元。不含自住房、不含公积金账户余额）

8. 各类贷款总余额（填写数字，单位：元。如无贷款请填 0）

9. 每月还贷款金额（填写数字，单位：元/月。已用公积金抵扣的部分不算；无贷款请填 0）

10. 每月固定生活开支（填写数字，单位：元/月。不含还贷款）

11. 每月预计还能拿出多少用于投资（填写数字，单位：元/月）

---

回复格式：只写有变化的项目，格式为「序号 答案」，空格或换行分隔。
示例：只改了年龄和收入 → 2 38 5 500000

说明：
- 没变的项目不用写，系统会沿用当前数据
- 序号 3、4（家庭、职业）：选字母
- 序号 5–11（金额）：填数字，单位是「元」，不要写"万"或"元"
- 回复「无变化」或「没有变化」→ 保留当前数据并直接确认

# Agent 须知（不对客）

- 发送前须调用 `profileRead()` 获取当前 `BasicInfo`，将 `{name}`、`{age}`、`{marital_status_label}`、`{occupation_label}`、`{annual_income_after_tax}` 等占位符替换为实际值后再发送给用户。`marital_status_label` 和 `occupation_label` 须转换为对客中文描述。
- **禁止**在对客消息中出现字段名（`basic_info`、`goal_type` 等）、问卷 ID、Hook 编号、库表名。
- 解析优先走 `parseChoiceFormat`（确定性多选格式）；该解析器支持部分输入——缺失的字段自动继承自现有 `BasicInfo` 记录，与用户输入合并后进入审视。
- 若用户回复「无变化」或「没有变化」，跳过解析，直接进入 §6.1.4 确认卡（沿用当前数据）。
- 选项字母到确切值的映射表由解析器代码维护，Agent 无须记忆或告知用户。
- 合并后须通过 §6.1.3 审视（Hook1 / Hook2 全量检查）；若 `financial_assets` 或 `monthly_investable` 变更，须额外触发 §6.2.2 跨组审视。
- 本问卷 **仅** 用于客户信息层修改（§6.4）；若用户意图落在某一组投资需求约束 → 路由至 §6.5，不得误发本问卷。
- 不得在对客正文中引用"示例/演示/样例基本情况"等内部话术。
