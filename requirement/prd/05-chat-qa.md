## 5.5 自由问答场景（`chat`）

### 模块说明

| 项 | 说明 |
|----|------|
| **做什么** | 像问理财朋友一样随便问；可发截图；复杂需求引导跳转其他 Tab |
| **入口** | Tab **「自由问答」**（新建对话默认） |
| **完成标志** | 当轮回答即可；**不**自动生成规划书/持仓/基金长报告 |
| **不做** | 写业务表；不要求数据库（SET-DB-01） |
| **编码锚点** | `simple_qa` · CH-10/12/18 · 空状态 §5.7 |

共有壳 → [05-chat-shared.md](./05-chat-shared.md)。Planner 短问 vs 场景流程 → [shared §5.6.2](./05-chat-shared.md)。四业务场景 → §6–§9。

### 5.5.1 模块目标

- **默认 Tab**；**不写业务表**（无 `scene_task` 写路径）  
- **短问**在本对话完成（共有 Planner · `simple_qa`）  
- **要进**需求梳理 / 资产配置 / 持仓 / 基金 **正式流程** → `cross_scene_handoff` + 跳转卡（§5.6.3）  
- 可带图（Vision · **五 Tab 共有** · §5.3.13）；**chat 专属**：截图仅 **建议** 改持仓（§5.3.16）  
- **能力介绍**：空状态 §5.7  
- **不要求 DB**（SET-DB-01）；推理 + 联网须就绪

#### 5.3.16 自由问答 · 识别持仓截图（CH-10）

| 步骤 | 规格 |
|------|------|
| 1 | Vision 解析 → 对话内 **建议**（表格/列表预览） |
| 2 | 助手 **先问**：「识别到以下持仓，**是否要去持仓分析更新**？您也可以继续在这里聊。」 |
| 3 | 确认卡：**去持仓分析更新** · **暂不，只聊聊**（同 §5.6.3 **HANDOFF-CONFIRM-01**） |
| 4 | **暂不** 或 **不点卡 · 直接发新消息** | 不跳转；卡片灰化；继续 `simple_qa` |
| 5 | **去更新**（= 点「前往」） | 解析 portfolio 对话 · `handoff_autostart` · 阶段条开跑 |

**禁止**：chat 内 **静默** `holdings_propose` / 写库。

### 5.7 能力介绍（「你能做什么」）

#### 5.7.1 展示位置

- **chat 空状态**：Logo 下常驻短文（可折叠 → 使用说明）  
- 任意 Tab 问「你能做什么」→ 均回复 §5.7.2  

#### 5.7.2 对客文案（定稿）

> 我是你的**理财助手**，可以帮你完成三件「有报告留痕」的事，也可以随时聊聊：
>
> 1. **投资规划** — 先完善投资需求与目标约束，再生成可解释的资产配置方案，并输出 **《投资规划书》**（完成标志）  
> 2. **持仓分析** — 录入持仓后做组合诊断；有目标方案时可给再平衡参考，并输出 **《持仓分析报告》**（完成标志）  
> 3. **基金解读** — 单只公募深度分析，并输出 **《单只基金分析报告》**（完成标志）  
> 4. **自由交流**（辅助）— 基金、市场、理财相关问题；**任意 Tab** 在已配置图片理解时均可发截图提问；复杂需求会引导你到上面三步  
>
> **范围说明**：规划、持仓与基金解读中的**基金产品仅覆盖中国公募基金**；**自由问答**可结合**公开网络信息**交流更广泛的话题。  
>
> 建议路径：**需求梳理 → 方案 → 持仓 → 基金**。你也可以只研究某一只基金，或先随便聊聊。  
> *以上均为信息参考，不构成投资建议。*
