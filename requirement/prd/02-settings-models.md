## 2.2 模型设置（全局）

### 模块说明

五类模型槽位（推理、深度推理、图片理解、联网、文本嵌入）及检测规则。**阻断逻辑** → [Hub §2.0.2](./02-settings.md)。Hub 对照 → [§2.0](./02-settings.md) ①。

路径：**设置 → 模型**（`/settings/models`）

### 2.2.1 五类模型槽位（本期）

| 槽位 | 用途 | 必填 | 检测可用性 |
|------|------|------|------------|
| **推理模型（快）** | Planner、简单问答、意图理解、阶段文案生成 | ✅ 必填 | ✅ |
| **深度推理模型** | 规划书、基金深度解读、复杂持仓分析 | 推荐 | ✅ |
| **图片理解模型** | **五场景**聊天发图、截图识别 | 用图片时必填 | ✅ |
| **联网搜索** | 全模式外网检索（§2.2.5） | ✅ **必填** | ✅ |
| **文本嵌入** | 基金 L2 语义检索（§2.2.7 · SET-EMB-01） | 用 L2 时必填 | ✅ |

> **本期不做**：**向量 Rerank** 槽位（KB-01：官方披露走本地 FTS；语义子库用 Supabase pgvector，见 [knowledge §9.2.0f](./09-fund-knowledge.md)，**不**依赖 Rerank）。

**联网搜索推荐组合**：推理 DeepSeek + 联网槽位独立配 Kimi（`$web_search`）；推理与联网均 Kimi 时可开「与推理模型相同」。

### 2.2.2 单槽位字段（3 项）

| 中文含义 | 字段名称 | 字段类型 | 字段说明 |
|----------|----------|----------|----------|
| 模型名称 | `model_name` | text | 请求 `model` 参数 |
| API 根地址 | `api_base_url` | text | OpenAI 兼容根地址 |
| API 密钥 | `api_key_encrypted` | text | 加密存储 · 落库见 §2.2.8 |

### 2.2.3 「与推理模型相同」（深度推理 · 图片理解 · 联网搜索 · 文本嵌入）

**深度推理**、**图片理解**、**联网搜索**、**文本嵌入** 四个槽位均采用同一交互：

| 开关状态 | 行为 |
|----------|------|
| **开启**（默认） | 该槽位三项只读，**复用推理模型**的模型名称 / API Base URL / API Key |
| **关闭** | 展开**独立三项**，供「推理 ≠ 多模态 / 联网 / embedding」组合 |

**图片理解**：推理为多模态（如 Kimi）时默认开启；DeepSeek 推理 + Kimi Vision 时须关闭并单独填写。

**联网搜索**：推理支持原生联网（如 Kimi `$web_search`）时默认开启；**推理不支持联网时**，检测会失败并提示关闭开关、单独配置（§2.2.5）。

**文本嵌入**：默认「与推理相同」；专用 embedding endpoint 时关闭并填独立模型名（如 `text-embedding-3-small`）。

### 2.2.4 检测可用性

#### 交互与按钮（Figma · P0）

| 项 | 规格 |
|----|------|
| **按钮位置** | **每个模型槽位**标题行或三项字段区域**右侧**，固定文案 **「检测可用性」**（五槽位） |
| **保存 vs 检测** | 页底 **「保存」**（或「确认」）**仅写入配置**（`model_settings`），**不**自动发起任何探测；用户须**主动点击**各槽位「检测可用性」 |
| **检测中** | 按钮 loading；状态行显示「正在检测，请稍候…」；禁止重复点击 |
| **结果展示** | 每个槽位标题下方固定 **一行检测状态**（见下「对客四态」）；**不**向用户展示任何英文字段名 |

**对客四态**（每个槽位独立 · 用户只看中文）：

| 状态 | 展示（示例） | 含义 |
|------|----------------|------|
| **待检测** | 灰色 · 「尚未检测」 | 已保存或未保存配置均可；用户还未点过本槽位「检测可用性」 |
| **检测中** | 蓝色 · 「正在检测…」 | 请求进行中 |
| **可用** | 绿色 · 「已通过检测，可正常使用」+ 次要行「上次检测：今天 14:32」 | 最近一次探测成功 |
| **不可用** | 红色 · 「检测未通过：{一句原因}」+ 次要行「上次检测：今天 14:32」 | 最近一次探测失败 |

**时间文案**：同一自然日 →「今天 HH:mm」；昨日 →「昨天 HH:mm」；更早 →「M月D日 HH:mm」。

**配置变更后**：用户修改该槽位任一项（模型名称 / 地址 / 密钥，或切换「与推理模型相同」）→ 状态**立即回退**为 **「尚未检测」**（或黄色提示「配置已变更，请重新检测」）。

**页内对客说明**（**设置 → 模型** 分组顶部 · 固定文案）：

> 建议您在填写完每个模型的配置后，**分别点击右侧「检测可用性」**，确认各模型能正常连接并返回预期结果后再开始对话。保存配置不会自动检测。

#### 阻断规则（运行时）

| 槽位 | 未检测 / 检测失败时 |
|------|---------------------|
| **推理模型** | **阻断全部对话**（所有 Tab） |
| **联网搜索** | **阻断全部对话**（§2.2.5） |
| **深度推理** | 不阻断日常对话；**阻断**规划书 / 基金深度报告 / 复杂持仓分析等 `deep_*` 任务 |
| **图片理解** | 不阻断纯文本；**阻断五 Tab 发图**与 `vision_parse` |
| **文本嵌入** | 不阻断日常对话；**阻断** L2 semantic index / search 与知识库「更新语义索引」 |

> **「与推理模型相同」**：检测时按**该槽位解析后的有效三项**发起探测；各槽位**仍须分别点击检测**。

#### 编码落库（实现参考 · 不对客展示）

服务端按槽位记录检测结果：

| 中文含义 | 字段名称 | 字段类型 | 字段说明 |
|----------|----------|----------|----------|
| 检测状态 | `check_status` | text | `unchecked` \| `checking` \| `passed` \| `failed` |
| 上次检测时间 | `last_checked_at` | timestamptz? | |
| 失败原因 | `last_error_message` | text? | 对客中文一句 |

表结构见 [§2.2.8 `model_settings`](./02-settings-models.md#228-字段规格model_settings)（每 `slot` 一行）。

#### 各槽位探测方案（须真实调用 · 非仅 ping URL）

实现：`POST /api/settings/models/test`，body `{ slot: 'reasoning' | 'deep' | 'vision' | 'web_search' | 'embedding' }`；服务端用**当前表单或已保存配置**组装 client，**必须收到厂商有效响应**才算成功。

| 槽位 | 探测动作 | 成功判定 |
|------|----------|----------|
| **推理模型** | `chat.completions` 发 **1 轮最小对话** | 2xx；响应含 assistant 文本；探针含 `PROBE_OK` |
| **深度推理** | 同上，使用深度槽位解析后的三项 | 同推理 |
| **图片理解** | 多模态 `chat.completions` + 内置探针图 | 2xx；响应含探针关键字 |
| **联网搜索** | **1 次最小检索**（`web_search` Tool 或 Kimi `$web_search`） | 2xx；**≥1 条**可解析检索结果 |
| **文本嵌入** | `embeddings.create` 发 **1 条短文本** | 2xx；返回 **非空向量** |

**大模型类探针（推理 · 深度推理）**

- **User**：`请只回复 exactly 这个词，不要加标点：PROBE_OK`
- **成功**：assistant 内容（trim 后）**包含** `PROBE_OK`

**图片理解探针图**

| 项 | 说明 |
|----|------|
| **路径** | `{APP_ROOT}/public/fixtures/vision-probe.png` |
| **内容要求** | 图内可见 **`VISION_PROBE`**（或固定数字探针，代码常量注明） |
| **User 文案** | `这张图里的探针文字是什么？只回复探针内容。` |

#### 对客反馈文案

| 结果 | 状态行主文案 |
|------|--------------|
| 检测成功 | 已通过检测，可正常使用 |
| 401/403 | 检测未通过：密钥无效或已过期，请检查 API Key |
| 404 / model | 检测未通过：地址或模型名称有误，请检查 API Base URL 与模型名称 |
| 超时 | 检测未通过：连接超时，请检查网络或稍后重试 |
| 探针不匹配 | 检测未通过：模型有回应，但结果异常，请确认是否选错模型 |
| 联网 · 与推理相同但不支持 | 检测未通过：当前推理模型不支持联网，请关闭「与推理模型相同」后单独配置 |

完整 Toast 示例见 [05-chat-shared.md](./05-chat-shared.md) §5.14、[05-chat-qa.md](./05-chat-qa.md) §5.14。

### 2.2.5 联网搜索槽位（必填 · 细则）

> 产品级摘要见 [00-overview §0.4–0.10 索引](./00-overview.md)；本节为 **设置域完整规格**。

| 项 | 说明 |
|----|------|
| **字段** | 与 §2.2.2 相同三项；「与推理相同」关闭后独立填写 |
| **检测** | 槽位右侧「检测可用性」；须真实检索（§2.2.4） |
| **典型配置** | Kimi + `$web_search`；或 OpenAI 兼容搜索端点 |
| **范围** | 作用于**全部** `conversation_type`；产品数据边界仍遵守 §0.8 |
| **与推理分工** | 本槽位只负责**检索**；最终回复由**推理模型**生成 |
| **推理未必支持联网** | 如 DeepSeek 等无原生联网 → 用户须 **关闭**「与推理相同」并 **单独填写**三项（常见：Kimi 检索 + DeepSeek 推理） |
| **阻断** | 未配置或未通过检测 → **不可发起任何对话**（与推理槽位同级） |

**基金相关场景的检索编排**（运行时 · 非设置 UI）：遵守 **KB-03 来源优先级**（[knowledge §9.2.0g](./09-fund-knowledge.md)）— L0 行情 → L1 披露 explore → L2 语义向量 → **L0–L2 均无有效结果时** L3 联网补充；**不做**全局 Rerank 槽位。

### 2.2.6 快慢模型路由

| 模型 | 典型任务 | 配置 |
|------|----------|------|
| **推理（快）** | Planner、`ExecutionPlan`、简单问答、确认卡片文案 | §2.2.1 推理槽位 |
| **深度推理** | 投资规划书生成、基金深度报告、多步持仓诊断 | 独立槽位；**「与推理模型相同」默认开启**（G-01） |

**对客说明**（深度推理槽位旁）：「用于生成规划书、深度基金报告等**更费脑**的任务。默认与日常对话用同一模型；若您有更强的模型，可关闭「相同」单独配置。」

Harness 按 `task_type` 路由：`planner` / `simple_qa` → 快；`deep_report` / `deep_analysis` → 深。

### 2.2.7 文本嵌入模型（SET-EMB-01 · 层内语义筛选 · P0）

> **主用途（EMB-FILTER-01）**：基金解析时，对 **L1 披露 explore**、**L2 FAQ**、**L3 联网** 各层已召回候选做 **层内 embedding 重排与截断**；**不**用于对话生成。  
> **子用途**：L2 小 FAQ 库（pgvector）索引写入时的向量化（[knowledge §9.2.0f](./09-fund-knowledge.md)）。

路径：**设置 → 模型 → 文本嵌入**（位于联网搜索下方）

| 项 | 说明 |
|----|------|
| **对客标题** | **文本嵌入模型** |
| **Helper** | 「用于基金解析时 L1/L2/L3 多路检索结果的语义筛选；与对话模型分开配置。」 |
| **启用开关** | **「启用基金解析语义筛选」** checkbox（`embedding_filter.enabled`）；**关闭**后 L1/L2/L3 退回关键词/FTS 排序，**不阻断**聊天 |
| **字段** | 嵌入模型名称 / API Base URL / API Key |
| **检测可用性** | 独立按钮；`embeddings.create` 探针 |
| **阻断** | 未配置 / 检测失败 / **用户关闭** → **不阻断**日常对话；层内重排降级为关键词分 |
| **落库** | `model_settings` 槽位 **`embedding`** + `app_settings.embedding_filter` |

**典型配置**：智谱 Embedding-3 或 OpenAI 兼容 `text-embedding-3-small`。
