# 货币基金基金解析报告 · 需求文档

> 状态：定稿  
> 适用范围：所有 `fund_type` 包含 `"货币型"` 的基金  
> 当前自选代表：天弘余额宝货币（000198）  
> 版本：v1.2

---

## 1. 产品定位

货币基金是**现金管理工具**，而非投资增值工具。报告核心目标是帮助用户理解产品的**流动性特征、收益水平与费率结构**，让用户在货币基金之间做出选择，而不是与股票/混合基金比较。

### 1.1 目标用户

- 有闲置资金、需要现金管理方案的投资者
- 在多个货币基金之间比较、选优的用户
- 需要了解费率、申赎规则、收益水平的理财者

### 1.2 与其他基金报告的核心差异

| 维度         | 通用模板（权益/混合/债券）      | 货币基金模板                       |
| ------------ | ------------------------------- | ---------------------------------- |
| 收益衡量     | 净值涨跌（%）、最大回撤         | 七日年化（%）、万份收益（元）      |
| 持有建议     | 长期配置、承受波动              | 现金管理、流动性优先               |
| 风险表述     | 波动风险、下行风险              | 流动性风险、信用风险               |
| 适合我的问题 | 风险等级/持有期限/费率/大类角色 | 不设此章——货币基金无"适不适合"问题 |
| 同类对比     | 同类排名可选                    | 不设此章（蛋卷 API 数据不可达）    |

### 1.3 设计原则

**货币基金报告必须完全独立实现，不走任何通用模板 / 共有模块逻辑。**

- 不调用通用模板的 `formatFkCiteSection`、`formatFootnoteDefinitions` 等函数。
- 不直接使用 `resolveAssetAllocationForCharts` 等面向权益/混合/债券基金的共有资产解析逻辑。
- 所有章节、引用、图表、数据解析均按货币基金自身需求单独实现。
- 其他基金类型仍可继续使用原有共有模块，本 PRD 只约束货币基金管线的实现。

## 2. 报告结构

```
第一章：产品介绍
第二章：基金经理
第三章：投资范围
第四章：费率结构
第五章：资产配置
第六章：专家观点
第七章：温馨提示
第八章：引用说明
```

**注意：不设"这只基金适合我吗"章节。** 货币基金作为现金管理工具，不需要用户做风险匹配判断。此章完全删除，不留决策清单、不留图表。

## 3. 逐章设计

### 3.1 第一章：产品介绍

一张 Markdown 表格，列出以下字段。所有字段均尽量展示，无数据时展示 `—`：

| 字段         | 说明                          | 数据来源                            | 是否必显     |
| ------------ | ----------------------------- | ----------------------------------- | ------------ |
| 基金代码     | 6 位数字代码                  | L0 `fund_basic`                     | ✅           |
| 基金简称     | 基金中文全称或简称            | L0 `fund_basic`                     | ✅           |
| 产品类型     | `type` 字段原值（如"货币型"） | L0 `fund_basic.type` → `type_label` | ✅           |
| 风险等级     | 风险评级（如"R1·低风险"）     | L0 `fund_basic.risk_level`          | ✅           |
| 基金规模     | 估算规模（亿元），含截止日期  | L3 `fund_share.aum_yi`              | 有数据则展示 |
| 投资风格     | `invest_type` 字段原值        | L0 `fund_basic.invest_type`         | 有数据则展示 |
| 起点金额     | 最低申购金额（万元）          | L0 `fund_basic.min_amount`          | 有数据则展示 |
| 预期收益率   | 基金宣告的预期收益区间        | L0 `fund_basic.exp_return`          | 有数据则展示 |
| 业绩比较基准 | 基准全称                      | L0 `benchmark_name`                 | 有数据则展示 |
| 成立日期     | 基金成立日                    | L0 `fund_basic.found_date`          | ✅           |
| 万份收益     | 最近交易日每万份收益（元）    | L3 `fund_money_fund_daily_em`       | 有数据则展示 |
| 七日年化     | 最近交易日七日年化（%）       | L3 `fund_money_fund_daily_em`       | 有数据则展示 |

**约束：**

- 产品类型 / 投资风格：展示 Tushare `fund_basic` 原始值，不做拼接或改写。

### 3.2 第二章：基金经理

| 字段     | 说明                 | 数据来源                     |
| -------- | -------------------- | ---------------------------- |
| 姓名     | 基金经理姓名         | L0 `fund_manager`            |
| 任职起始 | 上任日期             | L0 `fund_manager.begin_date` |
| 任职结束 | 离任日期（如已离任） | L0 `fund_manager.end_date`   |

**展示规则：**

- 若仅1人，单行列出。
- 若多人，表格逐行列出。
- 若有基金经理变更历史，展示最近1-2位。
- 无经理数据时，整节省略。

### 3.3 第三章：投资范围

**内容来源：** 最新季报的"投资范围"或"投资组合范围"原文。

**处理流程：**
1. 从最新季报原文中提取投资范围段落。
2. 经 LLM 总结为一句不超过 60 字的中文描述。

**展示规则：**

- 如果最新季报有投资范围内容，展示 LLM 总结。
- 如果缺失，则不展示这个章节，后面序号要往前。
- 用 Markdown 引用块（`>`）展示。
- 正文末尾只放引用标注（如 `[^1]`），完整引用信息集中在第八章"引用说明"。

### 3.4 第四章：费率结构

货币基金的费率是用户在不同货币基金之间选择的关键依据。

| 字段       | 说明                        | 数据来源                                  |
| ---------- | --------------------------- | ----------------------------------------- |
| 管理费     | 年管理费率（%）             | L0 `fund_basic.management` 等           |
| 托管费     | 年托管费率（%）             | L0 `fund_basic.custodian` 等            |
| 销售服务费 | 年销售服务费率（%）         | L0 `fund_basic` 相关字段 / L3 `xueqiu`  |
| 申购费     | 申购费率（货币基金通常为0） | 固定为 `-`                                |
| 赎回费     | 赎回费率（货币基金通常为0） | 固定为 `-`                                |

**展示规则：**

- 管理费、托管费、销售服务费优先使用 L0 数据。
- 申购费、赎回费固定展示为 `-`，不额外查找来源。
- 用 Markdown 表格展示。
- 附 LLM 一句话费率解读。

### 3.5 第五章：资产配置

**内容来源：** 最新季报中的"资产配置"表格数据。

**处理流程：**
1. 从最新季报原文中解析资产配置表格，提取资产类别与占净值比例。
2. 按资产类别汇总（如银行存款、债券、买入返售金融资产、同业存单、其他资产等）。
3. 记录数据有效时间（季报报告期）。

**展示规则：**

- 用 ECharts 饼图展示"资产类别 vs 占净值比例"。
- 图表下方展示数据有效时间。
- 正文末尾只放引用标注（如 `[^1]`），完整引用信息集中在第八章"引用说明"。
- 若最新季报无资产配置数据，本章不展示。

### 3.6 第六章：专家观点

**内容来源：** 该基金知识库下的全部 `expert_opinion` 文档。

**处理流程：**
1. 读取该基金知识库下所有 `expert_opinion` 类型文档。
2. 用 LLM 对全部观点进行综合概述。
**字数约束：**
- 最多 4 个要点。
- 目标总字数 ≤300 字（每条 ≤75 字含主题标签）。
- 若为完整表达语义需略微超出，LLM 可自行判断放行——**完整句子 > 精确字数**。
- 模板层在最后一个自然句号处截断，不产生半句话；若截断后只剩 ≤2 个完整要点，保留。

3. LLM 先识别每篇观点的主题标签（如货币政策、收益率展望、流动性判断、产品/平台优势、风险提示），再按主题聚合输出要点。
4. 若多篇观点重复同一判断，LLM 合并为一条，避免啰嗦。

**展示规则：**

- 每条要点以 `**主题：** 内容` 的形式呈现，结构清晰。
- 正文末尾只放引用标注（如 `[^2]`），完整引用信息集中在第八章"引用说明"。
- 若该基金知识库无专家观点文档，本章不展示。

### 3.7 第七章：温馨提示

固定免责文案：

> 本报告由系统根据公开信息与入库披露整理，仅供参考，不构成任何投资建议或收益承诺。基金过往业绩不预示未来表现，投资须自担风险。

### 3.8 第八章：引用说明

**设计原则：**

- 货币基金报告的引用说明**单独实现**，不复用通用模板的 `formatFkCiteSection` + `formatFootnoteDefinitions`。
- 全文只有本章展示完整文档名称与可点击地址；正文其他章节只放引用标注。
- 不区分"知识库文档"和"外部地址"两类表格，统一展示。

**展示格式：**

| 引用标注 | 文档名称 | 地址 |
| -------- | -------- | ---- |
| [^1]     | 天弘余额宝货币市场基金 · 2025年第三季度报告 | [点击查看](deep_link) |
| [^2]     | 2026年货币市场利率展望 | [点击查看](deep_link) |
| [^3]     | 天天基金-天弘余额宝货币A | [点击查看](external_url) |

**跳转规则：**

- 知识库文档地址：点击跳转 App 内基金知识库对应文章页。
- 外部地址：点击唤起系统默认浏览器打开。

**引用来源范围：**

- 最新季报（用于投资范围、资产配置）
- 专家观点文档（用于专家观点章节）
- 外部资讯（如有，用于补充信息）

费率、基金基本信息等来自 L0 授权行情数据，不在引用说明中单独列出。

## 5. 约束与规则

### 5.1 硬约束

| ID  | 约束                                       | 说明                                                             |
| --- | ------------------------------------------ | ---------------------------------------------------------------- |
| C1  | **禁用"近一年涨跌"和"近一年最大回撤"**     | 货币基金收益极小、回撤近乎为零，展示会误导用户将其与权益基金对比 |
| C2  | **禁用"这只基金适合我吗"整章**             | 货币基金是现金管理工具，无需用户做风险匹配判断                   |
| C3  | **无阅读指引**                             | 货币基金结构简单，阅读指引无增量信息                             |
| C4  | **无三句话读懂**                           | 货币基金定位、风险一目了然，三句话是冗余元素                     |
| C5  | **Chapter 2 不写"近一年收益约 X%"句式**    | 货币基金不用涨跌衡量，改为主述收益稳定性和同类比较               |
| C6  | **Chapter 3 不定性为"适合X年以上配置"**    | 改为现金管理角色                                                 |
| C7  | **禁用"赚不赚钱"和"长期持有"等权益化标题** | 货币基金不是"赚钱"工具，不讨论"持有期限"                       |
| C8  | **货币基金报告不走共有模块**               | 所有章节、引用、图表均单独实现                                   |

### 5.2 软约束

| ID  | 约束                                                      | 说明           |
| --- | --------------------------------------------------------- | -------------- |
| S1  | 产品介绍表按固定顺序排列字段（代码→简称→类型→…→万份收益） | 保证视觉一致性 |
| S2  | LLM 投资范围总结不超过 60 字                              | 避免冗长       |
| S3  | 费率有数据缺失时，只展示已有的行                          | 不留空行       |
| S4  | 专家观点综合概述不超过 300 字，分点输出                   | 便于快速阅读   |
| S5  | 资产配置图表优先使用季报原文细分类别                      | 更贴合货基报告 |
